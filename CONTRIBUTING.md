# Contributing to Base Node

## Code of Conduct

All interactions with this project are governed by our
[Code of Conduct][code-of-conduct].

By participating, you agree to uphold these standards. Violations may result in
a ban from this repository or, in severe cases, from all Base and/or Coinbase
projects.

[code-of-conduct]: https://github.com/coinbase/code-of-conduct

---

## Bug Reports

Before submitting a bug report, please make sure your issue
[has not already been reported][1]. It may already be fixed.

When opening a bug report, include the following:

- Clear and reproducible steps to trigger the issue
- The behavior you observed
- The behavior you expected and the reasoning behind it
- Any relevant logs, stack traces, or debugging output

Incomplete reports may be closed without investigation.

---

## Feature Requests

We welcome feature requests with or without pull requests.

When proposing a feature, please include:

- A clear description of the problem or need you are addressing
- The expected outcome or benefit
- Optional: ideas or approaches for implementation

The most effective way to get a feature added is to submit a pull request.

---

## Pull Requests

Before starting work on a pull request, we recommend first confirming that your
changes are likely to be accepted. You can do this by
[opening an issue][1] for discussion.

When submitting a pull request:

- Ensure your [commit messages are clear and well-written][2]
- Add tests for new features or behavior changes
- Follow existing project conventions and style
- Ensure all tests pass before submission

Pull requests that lack tests or clarity may be delayed or rejected.

---

## Support Requests

For security reasons, any communication that references Coinbase support tickets
will be ignored. Such requests may be redacted and locked.

All support-related inquiries must be submitted through
[our official support team][3].
